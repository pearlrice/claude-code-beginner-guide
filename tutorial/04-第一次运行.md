## 4.5 第一条对话

在 `>` 后面输入你想说的话，然后按回车：

```
> 你好，请帮我看看这个项目里有什么文件
```

Claude Code 会这样回应：

```
我会帮您查看项目里的文件。让我列出目录内容。

[调用 ls 工具]

当前项目目录包含以下内容：

tutorial/    images/    CLAUDE.md

这是一个 Claude Code 教程项目，包含教程文档和图片资源。
```

注意看——Claude Code **自己运行了命令**来查看文件。它不是在"猜"你的项目里有什么，而是真的在执行操作。

---

### CLAUDE.md —— 这是什么？

在上面的输出中，你看到了 `CLAUDE.md` 这个文件。现在来搞清楚它到底是什么。

**CLAUDE.md 就是项目的"使用说明书"**。Claude Code 每次启动时都会自动读取它，就像新员工入职第一天先看员工手册。

```
┌────────────────────────────────────────────────────────────┐
│               CLAUDE.md 的作用                              │
│                                                            │
│  没有 CLAUDE.md：                                           │
│    你："帮我写一篇产品介绍"                                    │
│    Claude Code：（先花 5 分钟自己探索项目）                    │
│    → "这是什么项目？要写成什么风格？有什么参考资料？"             │
│    → 效率低，结果可能跟你的预期不符                             │
│                                                            │
│  有 CLAUDE.md：                                             │
│    你："帮我写一篇产品介绍"                                    │
│    Claude Code：（启动时已读了 CLAUDE.md）                    │
│    → "这是一个面向年轻用户的消费品项目，                          │
│       写作风格要求活泼、有网感，参考文件在 docs/brand-guide.md"    │
│    → 直接按规范干活，产出符合预期                                │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

**CLAUDE.md 里写什么？** 一般包含这些内容：

| 内容   | 示例                                        |
| ---- | ----------------------------------------- |
| 项目介绍 | "这是一个面向年轻用户的护肤品牌官网"                    |
| 风格要求  | "文案要活泼、口语化，避免官方腔" |
| 文件结构 | "图片放在 images/ 文件夹，文档放在 docs/ 文件夹"   |
| 参考资料 | "产品介绍文案参考 docs/brand-guide.md 的第3-5段"      |
| 注意事项 | "不要修改 images/ 下的原始图片，只能复制后编辑"           |
| 常用操作 | "写完文档后，同步更新目录文件 00-目录.md"         |

---

### CLAUDE.md 不是 Claude Code 专属的

你可能以为 CLAUDE.md 是 Anthropic 公司发明的、只有 Claude Code 能用。**不是这样的。**

CLAUDE.md 的本质就是一个 **Markdown 格式的纯文本文件**，放在项目根目录。因为内容就是纯文字说明，所以任何 AI 编程工具都能读它：

| AI 工具                  | 支持的文件名                            |
| ---------------------- | --------------------------------- |
| **Claude Code**        | `CLAUDE.md`                       |
| **OpenAI Codex CLI**   | `AGENTS.md` 或 `CLAUDE.md`         |
| **Cursor**             | `.cursorrules` 或 `CLAUDE.md`      |
| **GitHub Copilot**     | `.github/copilot-instructions.md` |
| **Windsurf**           | `.windsurfrules`                  |
| **OpenCode**           | `AGENTS.md` 或 `CLAUDE.md`         |
| **Cline (VS Code 插件)** | `.clinerules`                     |

> 🎯 **核心理解**：CLAUDE.md 是一个**开放约定**，不是私有格式。你在 CLAUDE.md 里写的内容，换一个 AI 工具照样能用。Anthropic 只是率先推广了这个实践，业界正在向它靠拢。换个角度看，**你写的项目说明书，Codex、Cursor、Copilot 全都能用**——一份投入，多份回报。

---

### 补充：Markdown 是什么？为什么要用它？

CLAUDE.md 用的是 **Markdown** 格式（`.md` 后缀）。如果你还不会 Markdown，这里有一分钟速成。

**Markdown 就是用简单的符号来标记文字格式**。你写的是纯文本，但渲染出来有标题、列表、代码块等效果。

#### Markdown 基础语法（一分钟学会）

````
# 一级标题（对应 Word 里的"标题1"）
## 二级标题（对应 Word 里的"标题2"）
### 三级标题

**粗体文字**
*斜体文字*

- 无序列表项 1
- 无序列表项 2

1. 有序列表项 1
2. 有序列表项 2


代码块（用三个反引号包裹）：
```python
print("hello world")
```
````
```
行内代码：用一对反引号包裹，比如：  `这是一段代码`

嵌入链接：[对链接的说明/显示的文本](https://网址)

插入图片：![要显示的文本](/图片的路径)

表格：
| 列1 | 列2 |
|-----|-----|
| 内容 | 内容 |
```

#### 为什么 AI 工具都用 Markdown 而不是 Word 文档？

| 对比维度     | Markdown（`.md`） | Word（`.docx`）       |
| -------- | --------------- | ------------------- |
| 本质       | 纯文本文件           | 二进制压缩包（ZIP + XML）   |
| AI 能直接读吗 | ✅ 直接读，跟读 txt 一样 | ❌ 需要先解析 ZIP 再解析 XML |
| 人能直接看吗   | ✅ 记事本/任何编辑器都能打开 | ❌ 需要 Word 或兼容软件     |
| 能放代码吗    | ✅ 代码块原生支持       | ⚠️ 粘贴进去格式常乱         |
| git版本管理  | ✅ 纯文本，随时对比新旧版本  | ❌ 二进制，无法直接对比差异      |
| 文件大小     | 几 KB            | 几十到几百 KB（嵌入字体）      |
| 跨平台      | ✅ 手机/平板/任何系统都能看 | ⚠️ 手机可能排版错乱         |

> 📦 **一句话**：Markdown 让人类和 AI 都能高效阅读。Word 是给打印机排版用的，不适合 AI 和代码协作。这个教程本身就是用 Markdown 写的——你用记事本打开 `tutorial/04-第一次运行.md` 就能看到 `#`、`-`、` ``` ` 等原始符号，它们就是 Markdown 的标记。

## 4.6 两种对话模式

Claude Code 有两种使用模式：

### 模式一：交互式（Interactive Mode）

就是你刚才用的——输入 `claude` 启动，然后一问一答地对话：

```
PS K:\project> claude

> 帮我创建一个 index.html 文件
[Claude 创建文件]

> 把标题改成 Hello World
[Claude 修改文件]

> 提交这些改动
[Claude 执行 git commit]
```

**适合**：探索项目、多步开发任务、需要不断调整的需求。

### 模式二：一次性命令（Print Mode / Headless）

在启动时直接带任务，完成后自动退出：

```powershell
claude -p "帮我看看这个项目里有哪些安全漏洞"
```

`-p` = **p**rint mode，Claude 执行完任务后把结果打印出来，然后退出。

也可以把其他命令的输出"喂"给 Claude：

```powershell
# 把最近的日志丢给 Claude 分析
tail -200 app.log | claude -p "分析这些日志，找出异常"

# 把改动的文件列表给 Claude 审查
git diff main --name-only | claude -p "审查这些修改的文件"
```

**适合**：CI/CD 流水线、脚本自动化、快速问答。

```
┌────────────────────────────────────────────────────────────┐
│              两种模式对比                                   │
│                                                            │
│  交互式 (claude)               一次性 (claude -p)           │
│  ┌──────────────────┐        ┌──────────────────┐         │
│  │ 一问一答           │        │ 一句话交代完       │         │
│  │ 持续对话           │        │ 执行完就退出       │         │
│  │ 适合探索和开发     │        │ 适合脚本和CI       │         │
│  │ 有上下文记忆       │        │ 无持久上下文       │         │
│  └──────────────────┘        └──────────────────┘         │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

## 4.7 权限模式：让 Claude Code 做事前要不要问你？

Claude Code 在执行某些操作前会让你确认。这个"确认频率"由权限模式控制：

### 四种权限模式

```
危险 ←──────────────────────────────→ 安全
（不需要确认）              （每次都确认）

  YOLO      Accept Edits      Plan       Default
  全自动      接受编辑         只规划      默认
```

| 模式               | 别名   | Claude 能做什么       | 适合谁        |
| ---------------- | ---- | ----------------- | ---------- |
| **Default**      | 默认   | 读文件自动，写文件、跑命令需要确认 | 新手首选       |
| **Plan**         | 规划模式 | 只能读文件和规划，不能改任何东西  | 只是想先看看     |
| **Accept Edits** | 接受编辑 | 改文件自动，跑命令需要确认     | 有经验的用户     |
| **YOLO**         | 全自动  | 一切自动，不询问          | 老手，完全信任 AI |

### 切换权限模式

在对话中输入：

```
/permissions
```

会弹出权限配置界面。你也可以在启动时指定模式：

```powershell
claude --permission-mode plan
```

> ⚠️ **给新手的建议**：一开始用 **Default（默认）** 模式。每次 Claude 要做事情前都会问你"允许吗？"，你趁机看看它要干什么，这是学习的最好方式。

### 对话中遇到权限确认是什么样子？

```
Claude Code 想运行这个命令：

  mkdir new-feature

[Y] 允许  [N] 拒绝  [A] 全部允许  [D] 查看详情
```

- `Y`：这次允许
- `N`：这次拒绝
- `A`：本次对话中之后类似的都自动允许
- `D`：查看这个操作的具体内容

```powershell

或者直接按 `Ctrl+C` 两次。

Claude Code 会自动保存对话记录，下次进入同一项目时可以恢复。
