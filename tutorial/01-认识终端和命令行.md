# 第一章：认识终端和命令行

> **本章目标**：理解终端是什么，学会打开终端，掌握基本操作，为使用 Claude Code 做好准备。

---

## 1.1 终端是什么？

### 用一个生活类比来理解

想象你在手机上给朋友发微信：

| 操作     | 微信聊天         | 终端命令行                     |
| ------ | ------------ | ------------------------- |
| 打开聊天窗口 | 打开微信 App     | 打开终端程序                    |
| 输入文字   | 打字 "今天吃饭了吗？" | 打字 `ls`（显示文件列表）           |
| 收到回复   | 朋友回复 "吃了！"   | 电脑显示文件列表                  |
| 发送指令   | "帮我带杯奶茶"     | `mkdir my-project`（创建文件夹） |

**终端就是你和电脑"聊天"的窗口**。区别在于：你跟朋友聊的是自然语言，跟电脑聊的是**命令**——一种简短、精确的"电脑方言"。

```
┌─────────────────────────────────────────┐
│              终  端  窗  口               │
│                                         │
│  C:\Users\chris> ls                     │  ← 你输入的命令(列出当前目录)
│  tutorial/    images/    CLAUDE.md      │  ← 电脑的回复
│                                         │
│  C:\Users\chris> mkdir my-project       │  ← 再输入一条命令（创建文件夹）
│                                         │  ← 电脑创建了文件夹
│  C:\Users\chris> _                      │  ← 光标闪烁，等你下一条命令
│                                         │
└─────────────────────────────────────────┘
```

## 1.2 为什么需要终端来使用 Claude Code？

Claude Code 是一个**命令行工具**（CLI 工具），它天生就活在终端里。

> **CLI** = Command Line Interface（命令行界面）
> 就是我们刚才说的"打字聊天窗口"的正式名称。

你可能会问：**"为什么不用一个好看的 App 来操作呢？"**

实际上，Claude Code 也有桌面版和 VS Code 插件版。但因为一些特殊原因，**本教程推荐使用命令行版**：

| 版本           | 优点                       | 缺点                        |
| ------------ | ------------------------ | ------------------------- |
| **命令行版（推荐）** | 可直接用第三方 API Key，无需官方账号验证 | 界面是纯文字的                   |
| 桌面程序         | 界面精美                     | 必须通过官方账号验证，风控严格，国内用户基本不可用 |
| VS Code 插件   | 编辑器内使用                   | 同样需要账号验证                  |
| 网页版          | 无需安装                     | 功能受限                      |

> ⚠️ **关键提示**：Anthropic（Claude 的开发公司）对账号风控非常严格，中国大陆和许多地区用户即使付费购买了 Claude Pro 订阅也可能被无故封禁。而**命令行版 Claude Code 支持配置第三方 API 接口**（第八章详解），这就绕开了账号问题。这就是我们推荐命令行版的核心原因。

## 1.3 在 Windows 上打开终端

### Windows 10 / Windows 11 用户

你的电脑里已经自带了一个现代化终端：**Windows Terminal**（Windows 终端）。

它长这样：

```
┌──────────────────────────────────────────────────┐
│  ┌─ Windows Terminal ──────────────────────□ × ┐ │
│  │ PowerShell  │  CMD  │  +                    │ │ ← 标签页
│  │────────────────────────────────────────────  │ │
│  │                                              │ │
│  │  Windows PowerShell                          │ │
│  │  Copyright (C) Microsoft Corporation.        │ │
│  │                                              │ │
│  │  PS C:\Users\chris> _                        │ │ ← 命令行
│  │                                              │ │
│  └──────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────┘
```

### 三种打开方式（按推荐程度排列）

**方式一：右键菜单打开（最推荐，直接定位到当前文件夹）**

在任意文件夹的**空白处**，按住 `Shift` 键，同时点击鼠标右键，选择 **"在此处打开 PowerShell 窗口"** 或 **"在 Windows 终端中打开"**。

```
步骤图解：

  📁 我的项目文件夹/
  ├── src/
  ├── docs/
  └── README.md
       ↑
       在这个文件夹的空白区域：
       1. 按住 Shift 键不放
       2. 鼠标右键点击空白处
       3. 选择 "在 Windows 终端中打开"（Win11）
          或 "在此处打开 PowerShell 窗口"（Win10）
```
![[Pasted image 20260424175114.png|407]]


> 💡 **为什么推荐这种方式？** 因为终端打开后，自动定位到当前文件夹。这是使用 Claude Code 最重要的一步：**在项目文件夹中打开终端**。

**方式二：通过地址栏打开**

在任意文件夹的地址栏中，直接输入 `wt`（Windows Terminal 的缩写），然后按回车。

```
  ┌──────────────────────────────────────┐
  │ 📁 C:\Users\chris\my-project    [🔍]│ ← 文件管理器的地址栏
  │                                      │
  │   把地址栏内容删掉，输入: wt ↵        │
  │                                      │
  └──────────────────────────────────────┘
```

![[Pasted image 20260424175240.png]]

**方式三：通过开始菜单搜索**

按 `Win` 键，输入 "Terminal" 或 "终端"，点击打开。

> ⚠️ 注意：方法二和方法三打开后终端在用户主目录（`C:\Users\你的用户名`），需要手动 `cd` 到你的项目目录。（详情请看下文讲解）

## 1.4 终端里的"方言"：PowerShell vs CMD

打开 Windows Terminal 时，你会看到两种选择：

|       | PowerShell           | CMD（命令提示符）        |
| ----- | -------------------- | ----------------- |
| 提示符样子 | `PS C:\Users\chris>` | `C:\Users\chris>` |
| 现代化程度 | ✅ 现代、功能强             | ⚠️ 老旧（90年代风格）     |
| 推荐度   | ⭐⭐⭐ 推荐               | ⭐ 仅兼容老脚本时用        |

**本教程全部使用 PowerShell**（Windows Terminal 默认就是它）。

## 1.5 你的第一条命令

打开终端后，光标在闪烁——它在等你下命令。试试这几条：

### 第一条：看看你"站"在哪里

```powershell
pwd
```

这条命令让电脑告诉你当前在哪个文件夹里。相当于你在手机地图上点"我的位置"。

```
PS C:\Users\chris> pwd

Path
----
C:\Users\chris

PS C:\Users\chris>
```

### 第二条：看看周围有什么

```powershell
ls
```

或者用老式写法（两种都可以）：

```powershell
dir
```

这条命令列出当前文件夹里的所有文件和子文件夹。相当于你在房间里环顾四周，看看有什么东西。

```
PS C:\Users\chris> ls

    目录: C:\Users\chris

Mode                 LastWriteTime         Length Name
----                 -------------         ------ ----
d-----         2026/4/24     17:00                Desktop
d-----         2026/4/24     17:00                Documents
d-----         2026/4/24     17:00                Downloads
-a----         2026/4/24     16:30           1024 notes.txt
```

> 📖 **读法**：以 `d` 开头的是**文件夹**（目录），以 `-` 开头的是**文件**。

### 第三条：走进一个文件夹

```powershell
cd 文件夹名字
```

`cd` = **C**hange **D**irectory（切换目录），相当于你从一个房间走到另一个房间。

```powershell
PS C:\Users\chris> cd Documents
PS C:\Users\chris\Documents>    ← 注意这里变了！你已经在 Documents 里了
```

### 第四条：走回上一级

```powershell
cd ..
```

`..` 表示"上一级目录"，相当于从房间走回走廊。

### 第五条：创建新文件夹

```powershell
mkdir 新文件夹名
```

`mkdir` = **M**a**k**e **Dir**ectory（创建目录）。

```powershell
PS C:\Users\chris> mkdir my-claude-project

    目录: C:\Users\chris

Mode                 LastWriteTime         Length Name
----                 -------------         ------ ----
d-----         2026/4/24     17:30                my-claude-project
```

## 1.6 终端使用核心口诀

记住这几句，你就掌握了终端的日常使用：

| 你想做的事 | 命令 | 记忆方法 |
|-----------|------|---------|
| 我在哪？ | `pwd` | **P**rint **W**orking **D**irectory |
| 周围有什么？ | `ls` 或 `dir` | **L**i**s**t / **Dir**ectory |
| 走进某个文件夹 | `cd 名字` | **C**hange **D**irectory |
| 回到上一级 | `cd ..` | 两个点 = 上一级 |
| 创建文件夹 | `mkdir 名字` | **M**a**k**e **Dir**ectory |
| 清空屏幕 | `clear` 或 `Ctrl+L` | 字面意思 |
| 新建空文件 | `echo "" > 文件名` | 暂时用到的时候再学 |

## 1.7 工作区（项目目录）是什么

**工作区**就是 Claude Code 的"活动范围"。你把终端定位到哪个文件夹，Claude Code 就能看到那个文件夹里的所有文件，并且能在里面创建、修改、删除文件。

```
                                   ┌─────────────────┐
                                   │   Claude Code    │
                                   │  能看到的范围：   │
    📁 my-project/  ← 工作区       │  ├── src/        │
    ├── src/                       │  ├── docs/       │
    │   ├── app.js                 │  └── README.md   │
    │   └── style.css              │                  │
    ├── docs/                      │ 看不到的范围：    │
    │   └── guide.md               │  ✗ ../other/     │
    └── README.md                  │  ✗ C:\Windows\   │
                                   └─────────────────┘
```

> 🎯 **核心原则**：使用 Claude Code 前，一定要先用 `cd` 进入你的项目文件夹！

### 标准操作流程

```powershell
# 第 1 步：走进你的项目文件夹
cd C:\Users\chris\my-project

# 第 2 步：确认你站对地方了
pwd
# 输出：C:\Users\chris\my-project   ← 确认无误！

# 第 3 步：启动 Claude Code（第三章会讲）
claude
```
