> **本章目标**：掌握 Claude Code 的核心概念，理解它是怎么工作的，以及所有关键术语的含义。

## 5.1 Agent Loop（智能体循环）—— Claude Code 的"大脑"如何工作

这是理解 Claude Code 最重要的概念。**Agent Loop** 是 Claude Code 思考—行动—观察—再思考的循环过程。

### 用人话解释

想象你是一个厨师，要做一道菜：

```
传统 AI（ChatGPT 网页版）：
  你问："怎么做番茄炒蛋？"
  AI 答："先打蛋，再切番茄，然后下锅炒..."
  （它只是告诉你步骤，不会真的去做）

Claude Code（Agent Loop）：
  你："给我做番茄炒蛋"
  Claude Code：
    第 1 步 - 思考：我需要先看看冰箱里有番茄和鸡蛋吗 → 打开冰箱（读文件）
    第 2 步 - 观察：有番茄 3 个，鸡蛋 5 个
    第 3 步 - 思考：材料够，开始做。先打蛋 → 拿起碗和鸡蛋（运行命令）
    第 4 步 - 观察：蛋打好了
    第 5 步 - 思考：接下来切番茄 → 拿起刀（创建文件）
    第 6 步 - 观察：番茄切好了
    ...持续循环，直到完成
```

### Agent Loop 标准流程图

```
┌─────────────────────────────────────────────────────────────┐
│                    Agent Loop 循环                           │
│                                                             │
│   ┌──────────┐      ┌──────────┐      ┌──────────┐        │
│   │  观察     │ ──→  │  思考     │ ──→  │  行动     │        │
│   │ Observe  │      │  Think   │      │   Act    │        │
│   └──────────┘      └──────────┘      └──────────┘        │
│        ↑                                  │                │
│        └──────────────────────────────────┘                │
│                    回到观察（看结果）                         │
│                                                             │
│   具体例子：                                                  │
│   观察：读取 app.js，发现第 45 行报错                          │
│   思考：这里应该加一个空值检查                                 │
│   行动：用 Edit 工具修改第 45 行代码                           │
│   观察：修改后运行测试，测试通过                               │
│   思考：问题已解决，向用户报告                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

> 🎯 **关键理解**：Claude Code 不是"你问一次它答一次"，而是"你交代目标，它自己循环执行直到完成"。这就是 Agent（智能体）和 Chatbot（聊天机器人）的本质区别。

## 5.2 Context Window（上下文窗口）—— AI 的"短期记忆"

### 是什么

Claude Code 只能"记住"一定量的对话内容。这个容量就叫**上下文窗口**。

```
比喻：上下文窗口 = AI 的"眼前桌面"

  ┌──────────────────────────────────────┐
  │          AI 能看到的内容              │
  │                                      │
  │  📄 系统提示词                        │
  │  📄 CLAUDE.md 文件                    │
  │  📄 最近的对话                         │
  │  📄 刚读的文件内容                     │
  │  📄 你发的 Skill 内容                  │
  │                                      │
  │    ...太多内容就会超出 → 需要压缩       │
  │                                      │
  │  超出窗口的内容 → 🗑️ AI 看不到了       │
  │                                      │
  └──────────────────────────────────────┘
```

### 不同模型的上下文窗口

| 模型                | 上下文窗口                   |
| ----------------- | ----------------------- |
| Claude Opus 4.7   | 200K tokens（约 15 万英文单词） |
| Claude Sonnet 4.6 | 200K tokens             |
| Claude Haiku 4.5  | 200K tokens             |
| DeepSeek V3       | 128K tokens             |
| GPT-4o            | 128K tokens             |

> 💡 **Token 是什么？** Token 是 AI "阅读"的最小单位。粗略换算：1 个汉字 ≈ 2 tokens，1 个英文单词 ≈ 1.3 tokens。200K tokens 大约相当于一本 300 页的小说。

### 上下文满了怎么办？—— Compaction（自动压缩）

当对话太长、上下文快满了，Claude Code 会自动**压缩**：

```
压缩前（例如 180K tokens）：           压缩后（例如 30K tokens）：
┌──────────────────────────┐         ┌──────────────────────────┐
│ 用户: 帮我创建 index.html  │         │ [摘要] 用户要求创建了      │
│ Claude: 好的...           │         │ index.html 和 style.css  │
│ [完整的创建过程]           │   →     │ [摘要] 修复了4个lint错误   │
│ 用户: 修改标题...          │         │ [当前状态] 正在处理       │
│ Claude: 好的...           │         │ 登录功能，           │
│ [完整的修改过程]           │         │ 已修改 auth.js...        │
│ ...再加200轮对话           │         └──────────────────────────┘
└──────────────────────────┘
```

自动压缩后，Claude 保留了关键信息摘要，忘掉了对话细节。如果某个对话细节很重要，你需要重新告诉它。

> 💡 可以用 `/compact` 手动触发压缩，用 `/context` 查看当前上下文用量。

## 5.3 Tools（工具）—— Claude Code 的"手和眼睛"

Claude Code 不只是聊天——它有一组**工具**可以用来跟你的电脑交互。每个工具对应一种操作能力。

### 工具全家福

```
┌────────────────────────────────────────────────────────────┐
│                 Claude Code 的 7 大工具                      │
│                                                            │
│  ┌─────────┐ ┌──────────┐ ┌──────────┐ ┌───────────┐     │
│  │  Read   │ │  Write   │ │   Edit   │ │   Bash    │     │
│  │  读文件  │ │  写文件   │ │  改文件   │ │  跑命令    │     │
│  └─────────┘ └──────────┘ └──────────┘ └───────────┘     │
│                                                            │
│  ┌──────────┐ ┌───────────┐ ┌──────────────┐              │
│  │  Glob   │ │   Grep    │ │    Agent     │              │
│  │ 搜文件名  │ │ 搜文件内容 │ │ 派子代理干活  │              │
│  └──────────┘ └───────────┘ └──────────────┘              │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 每个工具的用途

| 工具 | 做什么 | 什么时候用 | 例子 |
|------|--------|-----------|------|
| **Read** | 读取文件内容 | 看代码、看配置、看文档 | `Read("src/app.js")` |
| **Write** | 创建/覆盖文件 | 建新文件 | `Write("index.html", "<!DOCTYPE html>...")` |
| **Edit** | 精确修改文件中的部分内容 | 改几行代码 | `Edit("app.js", old, new)` |
| **Bash** | 执行命令行 | 运行测试、安装依赖、Git 操作 | `Bash("npm run test")` |
| **Glob** | 按文件名模式搜索 | 找所有 .js 文件 | `Glob("**/*.test.js")` |
| **Grep** | 按内容搜索 | 找包含某段代码的文件 | `Grep("function login")` |
| **Agent** | 派子代理去干独立任务 | 同时探索多个目录 | `Agent(subagent_type="Explore")` |

### Claude Code 会选择用什么工具

你不需要手动调用工具——Claude Code 会根据你的需求自动选择：

```
你说："帮我在所有 js 文件里把 var 改成 const"

Claude Code 自动执行：
  1. 用 Grep 搜索所有包含 "var" 的 js 文件
  2. 用 Read 读取每个文件
  3. 用 Edit 把 "var" 替换为 "const"
  4. 用 Bash 运行 npm test 验证没改坏

全程你只发了一条中文指令。
```

## 5.4 Permissions（权限系统）—— 安全护栏

### 为什么需要权限系统

Claude Code 能执行操作系统命令、修改文件、访问网络——如果完全不加限制，它可能误操作导致数据丢失。

### 权限层级

```
┌─────────────────────────────────────────────────────┐
│              权限控制层级（由高到低）                    │
│                                                     │
│  Layer 1: 企业策略（Managed Settings）                 │
│  ↑ 企业管理员设定，个人无法覆盖                        │
│  │                                                   │
│  Layer 2: 用户设置（settings.json）                    │
│  ↑ 你在 ~/.claude/settings.json 中配置                │
│  │                                                   │
│  Layer 3: 项目设置（.claude/settings.json）            │
│  ↑ 项目级配置（对项目内所有人有效）                     │
│  │                                                   │
│  Layer 4: 运行时确认                                    │
│  ↑ 每次执行前的 Ask User Question                    │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 常用权限配置

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run *)",      // 自动允许 npm run 开头的命令
      "Bash(git diff *)",     // 自动允许 git diff
      "Bash(git status *)",   // 自动允许 git status
      "Read"                   // 自动允许读文件
    ],
    "deny": [
      "Bash(rm -rf *)",       // 禁止删除命令
      "Bash(git push *)"      // 禁止推送
    ]
  }
}
```

> 💡 在对话中随时输入 `/permissions` 来调整。

## 5.5 CLAUDE.md —— 项目的"使用说明书"

### 回顾：第四章已经见过它了

在第四章第一次对话中，Claude Code 列出项目文件时出现了 `CLAUDE.md`。当时我们简单了解了它是什么。现在来深入了解。

### 它到底是什么

`CLAUDE.md` 是一个放在项目根目录的纯文本文件。**Claude Code 在每次对话开始时都会自动读它**，作为理解项目的第一手资料。

把它想象成"入职指南"：新同事第一天上班，先看入职指南了解公司——Claude Code 也一样，启动先看 CLAUDE.md 了解项目。

### 真实案例：本教程的 CLAUDE.md

你现在读的这个教程项目里就有一个真实的 CLAUDE.md：

```markdown
# Claude Code 新手全方位教程 — 项目文档

## 项目目标
面向编程小白（纯新手），从零开始学习使用 Claude Code 的全方位中文教程。
参考 Notion 笔记风格，章节清晰，每步配有图示说明。

## 写作风格
- 面向纯新手，用生活化类比解释技术概念
- 每步骤配有说明
- 关键术语用 **粗体** 标注，附带括号解释
- 每章末尾有"本章小结"

## 文件结构
claude_tut/
├── CLAUDE.md                    # 项目计划
├── tutorial/
│   ├── 00-目录.md              # 总目录
│   ├── 01-认识终端和命令行.md
│   ├── ...（中间省略）
│   └── 08-第三方模型和计费.md
└── images/                      # 截图和图表

## 参考来源
- Claude Code 官方文档: https://code.claude.com/docs/en/overview
```

> 💡 **这份 CLAUDE.md 的实际效果**：当我在这个项目里让 Claude Code "帮我写一段新内容"，它会自动知道——这是给新手看的教程，语气要友好，用生活类比，末尾加小结。完全不用我每次重新解释。

### CLAUDE.md 里应该放什么

| 放什么  | 说明         | 来自本教程的真实例子                     |
| ---- | ---------- | ------------------------------ |
| 项目目标 | 这个项目是干什么的  | "面向新手用户，从零开始的 Claude Code 教程"  |
| 风格约定 | 有什么写作/编码规矩 | "用生活化类比，关键术语加粗，每章末尾小结"         |
| 文件结构 | 文件夹是怎么组织的  | "tutorial/ 下 00-08 共九个 .md 文件" |
| 注意事项 | 有什么坑不能踩    | "images/ 文件夹放截图"               |
| 参考来源 | 去哪里查更多资料   | "官方文档 code.claude.com/docs"    |

**核心原则**：写那些"每次换新同事，来干活都要重新交代一遍"的内容。

### CLAUDE.md 放在哪里（三个级别）

```
级别 1 — 用户级（全局生效）：
  C:\Users\你的用户名\.claude\CLAUDE.md
  ↑ 对所有项目都生效，适合放"你个人的偏好"（如：始终用中文回复）

级别 2 — 项目级（只对当前项目）：
  你的项目文件夹\CLAUDE.md
  ↑ 只对这个项目生效，适合放项目专属信息（推荐，可以提交到 Git 共享给团队）

级别 3 — 自动记忆（Claude 自己写的）：
  你的项目文件夹\.claude\memory\
  ↑ Claude Code 在工作中自动记录的零散信息，不需要你手动维护
```

> 🎯 **建议**：每个项目都建一个项目级 CLAUDE.md。内容不用多，哪怕只写 3 句话告诉 Claude "这是什么项目"也远好过没有。

## 5.6 Hooks（钩子）—— 自动化的"触发器"

### 是什么

Hooks 让你在 Claude Code 做某件事**之前**或**之后**自动执行一段脚本。

```
比喻：Hook = 门口的红外线感应器
  有人走过 → 门自动打开
  Claude 编辑了文件 → 自动运行代码格式化

比喻：Hook = 闹钟
  7:00 到了 → 闹钟响
  Claude 要提交代码了 → 自动先跑一遍测试
```

### 常见用法

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit",
        "hooks": [{
          "type": "command",
          "command": "npx prettier --write ${CLAUDE_TOOL_INPUT_FILE_PATH}"
        }]
      }
    ],
    "PreToolUse": [
      {
        "matcher": "Bash(git commit *)",
        "hooks": [{
          "type": "command",
          "command": "npm test"
        }]
      }
    ]
  }
}
```

| Hook 类型 | 触发时机 |
|-----------|---------|
| `PreToolUse` | 工具调用**之前** |
| `PostToolUse` | 工具调用**之后** |
| `Notification` | 用户空闲时发送通知 |
| `SessionStart` | 会话开始时 |
| `SessionEnd` | 会话结束时 |

## 5.7 MCP（模型上下文协议）—— 给 AI 接上"外部世界"

### 是什么

MCP（Model Context Protocol）是一个**开放标准**，让 AI 能连接到外部工具和数据源。

```
没有 MCP：
  Claude Code ← 只能读写本地文件

有 MCP：
  Claude Code ← MCP ──→ Jira（项目管理）
                   ──→ Google Drive（文档）
                   ──→ Slack（团队沟通）
                   ──→ 数据库（数据查询）
                   ──→ 任何自定义工具
```

### 新人不必深究

作为新手，你暂时不需要自己搭建 MCP 服务器。只需要知道有这个东西，以后看到 `.mcp.json` 文件知道是做什么的。

## 5.8 计费模式对比 —— 彻底搞懂 AI 是怎么收费的

这是新手最容易搞混的概念。我们用最直白的方式说清楚。

### 三种主流付费方式
订阅制 和按量付费

```
┌────────────────────────────────────────────────────────────┐
│                     AI 服务计费方式                          │
│                                                            │
│  ┌──────────────────┐  ┌──────────────┐  ┌──────────────┐ │
│  │  订阅制           │  │  API PAYG    │  │  混合制       │ │
│  │  Subscription    │  │  按量付费     │  │              │ │
│  ├──────────────────┤  ├──────────────┤  ├──────────────┤ │
│  │ 付月费，随便用    │  │ 用多少付多少  │  │ 月费+超额按量  │ │
│  │ 有次数上限        │  │ 无次数上限    │  │              │ │
│  ├──────────────────┤  ├──────────────┤  ├──────────────┤ │
│  │ 代表：            │  │ 代表：        │  │ 代表：        │ │
│  │ ChatGPT Plus     │  │ OpenAI API   │  │ Claude Max   │ │
│  │ ($20/月)         │  │ DeepSeek API │  │              │ │
│  │ Claude Pro       │  │ Anthropic API│  │              │ │
│  │ ($20/月)         │  │              │  │              │ │
│  └──────────────────┘  └──────────────┘  └──────────────┘ │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 用生活类比

| 计费方式              | 生活类比                      | 适合谁          |
| ----------------- | ------------------------- | ------------ |
| 订阅制（ChatGPT Plus） | 自助餐：付固定价格随便吃，但短时间内吃太多会被限制 | 每天用，但用量不会特别大 |
| API PAYG（按量付费）    | 点菜：吃一个菜付一个菜的钱             | 不确定用量，或用量波动大 |
| 混合制（Claude Max）   | 自助餐 + 可以额外加菜              | 重度用户         |
|                   |                           |              |

### API 计费到底怎么算？

API 的计费单位是 **token**（前面解释过）。不同模型价格不同：

**以 Claude Sonnet 为例**（价格会变动，仅供参考）：
- 输入（你发给 AI 的内容）：$3 / 百万 tokens
- 输出（AI 回复的内容）：$15 / 百万 tokens

**算一笔账**：
```
如果你用 Claude Code 写了一天的代码：
- 你发的消息 + 项目文件内容（输入）：约 200K tokens = $0.0006
- AI 的回复（输出）：约 50K tokens = $0.00075
- 单次对话消耗：约 $0.001（0.1 美分）

一天几百轮对话：大概 $1-3 美元
一个月重度使用：大约 $30-100 美元
```

> 💡 **关键区别**：API 按量付费的好处是——如果你一个月不用，一分钱不花。订阅制你付了月费，不用也扣钱。

### 三种方案的完整对比

| 对比维度 | ChatGPT Plus 订阅 | Claude Pro 订阅 | API PAYG（Claude Code 推荐） |
|---------|------------------|-----------------|---------------------------|
| 付费方式 | $20/月（或 $200/月 Pro） | $20/月（或 $100/月 Max） | 按实际使用量 |
| 使用限制 | 有次数上限（如 50 条/3 小时） | 有次数上限 | 无，有钱就能用 |
| 能用于 Claude Code | ❌ 不能 | ✅ 可以 | ✅ 可以 |
| 国内支付 | 需海外信用卡 | 需海外信用卡 | 可用支付宝（DeepSeek 等） |
| 封号风险 | 低 | **高**（风控严格） | 低（第三方平台） |
| 不用时的费用 | 照扣月费 | 照扣月费 | 0 元 |

> 🎯 **结论**：对于国内用户，**API 按量付费是相对实际的选择**。配合第三方平台（如 DeepSeek）的 API，可以用支付宝充值，按量付费，不封号。

或者你也可可以选择购买国内厂商各种coding plan 或token plan 套餐，然后接入他们套餐指定的api地址
